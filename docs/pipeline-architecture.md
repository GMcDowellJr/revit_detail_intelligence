# Pipeline Architecture

## Overview

The pipeline converts model geometry into comparable feature representations and evaluates similarity between construction conditions.

---

## Step 1 — Section Candidate Generation

Input:

- user-selected exterior wall face

Process:

- scan a vertical band of the model
- detect changes in element composition
- generate candidate section views

Output:

- list of candidate section locations

---

## Step 2 — Geometry Extraction

For each section:

- extract curves generated by model elements intersecting the section plane
- convert curves to endpoints in view coordinates

---

## Step 3 — Endpoint Clustering

Endpoints are grouped using spatial clustering.

Clusters represent localized areas where:

- multiple elements intersect
- geometry changes direction
- structural complexity increases

These clusters indicate areas likely requiring detail.

---

## Step 4 — Provenance Extraction

Each endpoint is associated with the element that generated it.

Collected attributes may include:

- category
- family
- type
- geometric role

These attributes become semantic tokens.

---

## Step 5 — Geometry Fingerprint

Cluster geometry is summarized into a normalized fingerprint.

This fingerprint captures structural relationships between endpoints while ignoring small dimensional variation.

---

## Step 6 — Detail Indexing

Existing details and drafting views are processed into the same feature representation.

The resulting feature vectors are stored in an index.

---

## Step 7 — Similarity Matching

Feature vectors are compared using similarity metrics.

The system identifies the closest matches within:

- project details
- library details

---

## Step 8 — Confidence Ranking

Matches are ranked according to weighted similarity scores.

Confidence categories are assigned based on calibrated thresholds.
